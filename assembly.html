Why Learn Assembly?

Compiler Output

Understand what gcc -O2 actually produces. Read disassembly like prose, not hieroglyphics.

Debugging

When the debugger drops you into disassembly, you need to know what you're looking at. Segfaults make sense.

Performance

Know why one loop is 10x faster. Understand cache lines, branch prediction, and SIMD at the instruction level.

Security Research

Buffer overflows, ROP chains, reverse engineering malware — all require reading assembly fluently.

You don't write it daily, but reading it makes you 10x better at everything else.

The Mental Model

The CPU runs this cycle billions of times per second. Every instruction you write goes through it.

High-Level (C)

int x = a + b;

Assembly (x86-64)

mov  eax, [a]      ; load a into eax
add  eax, [b]      ; eax = eax + b
mov  [x], eax      ; store result into x

CPU has registers (fast storage on the chip), memory (RAM, slow), and a stack (structured memory for function calls).

Memory Model

High addresses (0x7FFF...)
+-------------------------+
|        Stack            |  ↓ Grows downward
|  (local vars, frames)   |
+-------------------------+
|          |              |
|          v              |
|     (free space)        |
|          ^              |
|          |              |
+-------------------------+
|        Heap             |  ↑ Grows upward
|  (malloc, new)          |
+-------------------------+
|        BSS              |  Uninitialized globals (zeroed)
+-------------------------+
|        Data             |  Initialized globals & statics
+-------------------------+
|        Text             |  Executable code (read-only)
+-------------------------+
Low addresses (0x0040...)

Virtual memory: each process sees its own 48-bit address space (~256 TB). The OS maps virtual pages to physical RAM via page tables. Stack typically starts near 0x7FFFFFFFFFFF and grows down.

Your First Program

; hello.asm - Hello World in x86-64 Linux (NASM syntax)

section .data
    msg db "Hello, World!", 10   ; string + newline (10 = '\n')
    len equ $ - msg              ; length = current pos - start

section .text
    global _start

_start:
    ; write(1, msg, len)  -- syscall number 1
    mov rax, 1           ; syscall: sys_write
    mov rdi, 1           ; fd: stdout
    mov rsi, msg         ; buffer address
    mov rdx, len         ; number of bytes
    syscall              ; invoke kernel

    ; exit(0)  -- syscall number 60
    mov rax, 60          ; syscall: sys_exit
    xor rdi, rdi         ; exit code 0
    syscall

# Build and run:
$ nasm -f elf64 hello.asm -o hello.o
$ ld hello.o -o hello
$ ./hello
Hello, World!

; Real-world examples

; Struct field access: point->y where y is at offset 8
mov eax, [rdi+8]           ; rdi = struct pointer, offset 8

; Array indexing: arr[i] where each element is 4 bytes
mov eax, [rbx+rcx*4]      ; rbx = base, rcx = index

; 2D array: matrix[row][col] with 16-byte rows
lea rax, [rdi+rsi*4]      ; compute address
mov edx, [rax+rcx*4]      ; load element

; Stack local variable
mov eax, [rbp-12]          ; 3rd local int variable

The Stack

; How push/pop work:
;   push rax  ==  sub rsp, 8 ; mov [rsp], rax
;   pop  rax  ==  mov rax, [rsp] ; add rsp, 8

; Stack grows DOWN (high addr -> low addr)

; Function prologue (standard stack frame):
my_function:
    push rbp            ; save caller's base pointer
    mov  rbp, rsp       ; set up our frame pointer
    sub  rsp, 32        ; allocate 32 bytes for locals

    ; ... function body ...
    ; local vars at [rbp-8], [rbp-16], etc.

    ; Function epilogue:
    leave               ; equivalent to: mov rsp, rbp; pop rbp
    ret                 ; pop return address, jump to it

Stack during function call:

  High addresses
  +--------------------+
  | caller's locals    |
  +--------------------+
  | return address     |  ← pushed by CALL
  +--------------------+
  | saved rbp          |  ← pushed by PUSH RBP
  +--------------------+  ← rbp points here
  | local var 1        |  [rbp - 8]
  +--------------------+
  | local var 2        |  [rbp - 16]
  +--------------------+
  | local var 3        |  [rbp - 24]
  +--------------------+  ← rsp points here
  Low addresses

Control Flow

C: if / else

if (x > 10) {
    result = 1;
} else {
    result = 0;
}

x86-64 Assembly

    cmp  eax, 10       ; compare x with 10
    jle  .else         ; jump if x <= 10
    mov  ecx, 1        ; result = 1
    jmp  .end
.else:
    mov  ecx, 0        ; result = 0
.end:

C: for loop

int sum = 0;
for (int i = 0; i < n; i++) {
    sum += arr[i];
}

x86-64 Assembly

    xor  eax, eax      ; sum = 0
    xor  ecx, ecx      ; i = 0
.loop:
    cmp  ecx, edx      ; i < n?
    jge  .done         ; if not, exit
    add  eax, [rdi+rcx*4] ; sum += arr[i]
    inc  ecx           ; i++
    jmp  .loop
.done:

Functions & the Call Stack

C: Recursive Factorial

int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

x86-64 Assembly

factorial:
    push rbp
    mov  rbp, rsp
    push rbx             ; save callee-saved
    mov  ebx, edi        ; save n in rbx

    cmp  edi, 1
    jle  .base           ; if n <= 1, return 1

    lea  edi, [rbx-1]    ; arg = n - 1
    call factorial        ; rax = factorial(n-1)
    imul eax, ebx        ; rax = n * factorial(n-1)
    jmp  .done

.base:
    mov  eax, 1          ; return 1

.done:
    pop  rbx             ; restore callee-saved
    pop  rbp
    ret

call pushes the return address (next instruction's rip) onto the stack, then jumps to the target. ret pops the return address and jumps back. Each recursive call adds a new stack frame.

Working with C

Calling C from Assembly

; printf_demo.asm
section .data
    fmt db "Value: %d", 10, 0

section .text
    global main
    extern printf

main:
    push rbp
    mov  rbp, rsp

    ; printf("Value: %d\n", 42)
    lea  rdi, [rel fmt]   ; 1st arg: format
    mov  esi, 42          ; 2nd arg: value
    xor  eax, eax         ; 0 vector args
    call printf

    xor  eax, eax         ; return 0
    pop  rbp
    ret

Calling Assembly from C

; add.asm -- a function callable from C
section .text
    global add_numbers

add_numbers:            ; int add_numbers(int a, int b)
    lea  eax, [rdi+rsi] ; return a + b
    ret

// main.c
extern int add_numbers(int a, int b);

int main() {
    int result = add_numbers(3, 7);
    // result == 10
}

# Build:
$ nasm -f elf64 add.asm -o add.o
$ gcc main.c add.o -o demo

SIMD / Vectorization

Single Instruction, Multiple Data — process 4/8/16 values with one instruction.

Scalar (1 at a time)

; Add 4 floats, one by one:
movss  xmm0, [rdi]     ; load float 0
addss  xmm0, [rsi]     ; add
movss  [rdx], xmm0     ; store
; ...repeat 3 more times
; Total: 12 instructions

SIMD (4 at once)

; Add 4 floats simultaneously:
movaps xmm0, [rdi]     ; load 4 floats
addps  xmm0, [rsi]     ; add all 4
movaps [rdx], xmm0     ; store all 4
; Total: 3 instructions
ExtensionRegister WidthFloats per OpDoubles per Op
SSE (1999)128-bit: xmm0-xmm1542
AVX (2011)256-bit: ymm0-ymm1584
AVX-512 (2016)512-bit: zmm0-zmm31168

Key instructions: addps, mulps, divps, sqrtps (packed single-precision). Compilers auto-vectorize loops with -O2 / -O3 when they can prove it's safe.

Common Patterns in Disassembly

; Array access: arr[i]  (int arr[], index in ecx)
mov eax, [rdi + rcx*4]          ; base + index * sizeof(int)

; Struct field: point->y  (struct at rdi, y at offset 8)
mov eax, [rdi + 8]

; String length (like strlen):
.loop:
    cmp  byte [rdi], 0           ; check for null terminator
    je   .done
    inc  rdi                     ; next char
    inc  rax                     ; count++
    jmp  .loop

; memcpy-like pattern (rep movsb):
    mov  rdi, dest               ; destination
    mov  rsi, src                ; source
    mov  rcx, count              ; byte count
    rep  movsb                   ; copy rcx bytes from [rsi] to [rdi]

; Virtual function call (C++ vtable):
    mov  rax, [rdi]              ; load vtable pointer from object
    call [rax + 16]              ; call 3rd virtual method (index 2 * 8)

; Switch/jump table:
    cmp  eax, 5                  ; bounds check
    ja   .default
    lea  rcx, [rip + .jtable]    ; load jump table address
    movsxd rax, dword [rcx + rax*4]  ; load offset
    add  rax, rcx                ; compute target
    jmp  rax                     ; dispatch

Security: Buffer Overflows

; Vulnerable C code:
; void vuln(char *input) {
;     char buf[64];
;     strcpy(buf, input);    // No bounds check!
; }

; Stack layout during vuln():
; +--------------------+  High addresses
; | return address     |  ← attacker overwrites this
; +--------------------+
; | saved rbp          |  ← overwritten too
; +--------------------+
; | buf[56..63]        |
; | ...                |  ← overflow goes UP the stack
; | buf[0..7]          |
; +--------------------+  Low addresses (rsp)

; If input is longer than 64 bytes, it overwrites:
;   1. saved rbp (frame pointer)
;   2. return address (controls execution!)
;   3. Attacker redirects execution to their shellcode

Stack Canaries

Random value placed before return address. If modified, program aborts. gcc -fstack-protector

ASLR

Randomize stack/heap/library addresses each run. Attacker can't predict where to jump.

NX Bit (W^X)

Memory is writable OR executable, never both. Prevents executing injected shellcode on the stack.

ROP Chains

Modern exploit technique: chain existing code "gadgets" ending in ret to bypass NX. Defeated by CFI.

Performance: What Assembly Teaches You

Cache Lines (64 bytes)

CPU loads memory in 64-byte chunks. Sequential access = fast. Random access = cache miss = ~100 cycles penalty.

; Fast: sequential (cache-friendly)
mov eax, [rdi]        ; ~4 cycles (L1 hit)
mov eax, [rdi+4]      ; same cache line!

; Slow: random stride
mov eax, [rdi]        ; ~4 cycles
mov eax, [rdi+4096]   ; ~100+ cycles (cache miss)

Branch Prediction

CPU predicts which way a branch goes. Misprediction = ~15 cycle pipeline flush.

; Predictable: mostly taken or not taken
cmp eax, 0            ; 99% of time eax > 0
je  .rare_case        ; branch predictor learns

; Unpredictable: random 50/50
cmp eax, ebx          ; random comparison
jl  .path_a           ; pipeline stalls often

Instruction Pipelining

CPU executes multiple instructions simultaneously. Data dependencies stall the pipeline.

; Bad: each depends on previous
add rax, rbx    ; rax ready
add rax, rcx    ; STALL: waiting for rax
add rax, rdx    ; STALL: waiting for rax

; Good: independent instructions
add rax, rbx    ; these can run
add rcx, rdx    ; in parallel!
add r8,  r9     ; (out-of-order execution)

Memory Alignment

Unaligned access may cross cache line boundaries. Keep structs aligned to avoid split loads.

; Aligned (fast): address divisible by size
mov rax, [rdi]     ; rdi = 0x1000 (aligned)

; Unaligned (slower or fault):
mov rax, [rdi]     ; rdi = 0x1003 (misaligned)
; SIMD requires 16-byte alignment for movaps
