Tab Completion

How It Works

  • Predicts your next edit as you type
  • Shows ghost text (gray) inline
  • Press Tab to accept
  • Press Esc to dismiss
  • Learns from your editing patterns

What Makes It Special

  • Multi-line — predicts entire blocks, not just one line
  • Edit prediction — suggests changes to existing code, not just insertions
  • Context-aware — reads surrounding files, imports, types
  • Cursor prediction — predicts where you will move next

Pro tip: After accepting a Tab completion, press Tab again — Cursor often chains multiple related edits. It knows the next logical place to edit.

Inline Edit (Ctrl+K)

Select code (or place cursor on a line), press Ctrl+K, type what you want.

Example Prompts

  • "Add error handling for the API call"
  • "Convert this to async/await"
  • "Add TypeScript types"
  • "Make this function pure"
  • "Optimize this SQL query"
  • "Add input validation"

Tips

  • Select the exact code you want changed
  • Be specific about the desired outcome
  • Ctrl+Shift+K for terminal command generation
  • Shows diff preview before applying
  • Accept with Enter, reject with Esc
  • Works in any file type

No selection? Ctrl+K with no selection generates new code at cursor position. Great for "write a function that..."

Chat Panel (Ctrl+L)

Capabilities

  • Ask questions about your codebase
  • Generate new code with full context
  • Debug errors — paste stack traces
  • Explain complex code sections
  • Plan architecture and refactoring
  • Apply code directly from chat to editor

Context Controls

  • @file — reference a specific file
  • @folder — reference a directory
  • @code — reference a symbol/function
  • @web — search the web for context
  • @docs — reference documentation
  • @git — reference git history/diff

Apply button: Chat generates code? Click "Apply" to insert it directly into the right file at the right location.

Composer (Ctrl+I)

Cursor's most powerful feature — an AI agent that edits multiple files at once.

Multi-file Edits

Describe a feature: "Add user authentication with JWT, including routes, middleware, and tests." Composer edits all relevant files.

Agentic Mode

Composer can run terminal commands, read files, and iterate on errors automatically. Like having a pair programmer.

Checkpoints

Every Composer action creates a checkpoint. Review changes file-by-file, accept or revert individual files.

Iterate

Not satisfied? Follow up: "Also add rate limiting" or "Use bcrypt instead of argon2." Composer builds on previous context.

Best for: New features, refactoring across files, adding tests, migrations, boilerplate generation.

Codebase Indexing

How It Works

  • Cursor creates embeddings of your entire codebase
  • Indexes on project open (automatic)
  • Re-indexes on file changes
  • Enables semantic search across all files
  • Powers @codebase queries

Optimize Indexing

  • Use .cursorignore to exclude files
  • Exclude: node_modules, dist, build, .git
  • Exclude: large data files, binary assets
  • Check indexing status in Settings → Features
  • Re-index manually if results seem stale

Cursor Rules

Custom instructions that shape how Cursor AI responds. Like a system prompt for your project.

# .cursor/rules (project-level rules)

You are an expert in TypeScript, React 19, and Next.js 15 App Router.

Key principles:
- Use functional components with TypeScript interfaces
- Prefer server components; use "use client" only when needed
- Use Tailwind CSS for styling, no inline styles
- Follow the project structure in src/
- Write tests for all new utilities using Vitest
- Use Zod for runtime validation
- Prefer named exports over default exports
- Error messages should be user-friendly

Naming conventions:
- Components: PascalCase (UserCard.tsx)
- Utilities: camelCase (formatDate.ts)
- Types: PascalCase with prefix (type UserProps)
- API routes: kebab-case (/api/user-profile)

Location: .cursor/rules for project rules. Settings → General → Rules for global rules.

Debugging with Cursor

Paste Error Messages

Copy a stack trace into Chat (Ctrl+L). Cursor reads the referenced files and explains the fix.

Terminal Integration

Errors in the terminal? Cursor detects them and offers "Fix with AI" suggestions automatically.

Lint Errors

Hover over red squiggles → Quick Fix → "Fix with Cursor." AI understands ESLint, TypeScript errors.

Explain Code

Select confusing code, Ctrl+L, ask "What does this do?" Get a plain-English explanation with context.

Workflow: See error → Ctrl+L → Paste error → AI diagnoses → Click Apply → Fixed. 30 seconds.

Privacy & Security

Privacy Mode

  • Enable in Settings → General → Privacy Mode
  • Code is not stored on Cursor servers
  • Not used for training AI models
  • SOC 2 compliant
  • Essential for proprietary/enterprise code

Security Best Practices

  • Enable Privacy Mode for work projects
  • Use .cursorignore for sensitive files
  • Don't paste API keys into chat
  • Review AI-generated code before committing
  • Use your organization's approved models
  • Check compliance requirements (HIPAA, GDPR)
