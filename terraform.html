What is Terraform?

  • An open-source Infrastructure as Code (IaC) tool by HashiCorp
  • Define cloud resources in declarative configuration files (HCL)
  • Supports AWS, Azure, GCP, Kubernetes, and 3000+ providers
  • Manages the full lifecycle: create, update, and destroy infrastructure
  • Tracks resource state to determine what changes are needed

Key Features

Declarative Syntax

Describe the desired end state — Terraform figures out how to get there

Execution Plans

terraform plan shows exactly what will change before applying

Resource Graph

Builds a dependency graph and creates/modifies resources in parallel

State Management

Tracks the real-world state of your infrastructure for accurate diffs

HCL — HashiCorp Configuration Language

# Provider configuration
provider "aws" {
  region = "us-east-1"
}

# Resource definition
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  tags = {
    Name        = "web-server"
    Environment = "production"
  }
}

# Output values
output "public_ip" {
  value = aws_instance.web.public_ip
}

Variables & Locals

# variables.tf
variable "environment" {
  description = "Deployment environment"
  type        = string
  default     = "dev"
}

variable "instance_count" {
  type    = number
  default = 2
}

variable "allowed_cidrs" {
  type    = list(string)
  default = ["10.0.0.0/8"]
}

# locals
locals {
  name_prefix = "${var.project}-${var.environment}"
  common_tags = {
    Environment = var.environment
    ManagedBy   = "terraform"
  }
}

# Usage
resource "aws_instance" "app" {
  count         = var.instance_count
  instance_type = var.environment == "prod" ? "t3.large" : "t3.micro"
  tags          = local.common_tags
}

State Management

  • Terraform stores the current state of your infrastructure in a state file
  • Compares state vs config to determine what needs to create, update, or delete
  • Local state: terraform.tfstate (default, not recommended for teams)
  • Remote state: S3, GCS, Azure Blob, Terraform Cloud (recommended)
# Remote state backend (S3 + DynamoDB for locking)
terraform {
  backend "s3" {
    bucket         = "my-terraform-state"
    key            = "prod/terraform.tfstate"
    region         = "us-east-1"
    dynamodb_table = "terraform-locks"
    encrypt        = true
  }
}

Modules

Modules are reusable packages of Terraform configuration:

# Using a module from the Terraform Registry
module "vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "5.1.0"

  name = "my-vpc"
  cidr = "10.0.0.0/16"

  azs             = ["us-east-1a", "us-east-1b"]
  private_subnets = ["10.0.1.0/24", "10.0.2.0/24"]
  public_subnets  = ["10.0.101.0/24", "10.0.102.0/24"]

  enable_nat_gateway = true
  single_nat_gateway = true
}

# Using a local module
module "app_server" {
  source        = "./modules/ec2-instance"
  instance_type = "t3.micro"
  subnet_id     = module.vpc.private_subnets[0]
}

Data Sources

Data sources let you read existing infrastructure without managing it:

# Look up the latest Amazon Linux 2 AMI
data "aws_ami" "amazon_linux" {
  most_recent = true
  owners      = ["amazon"]

  filter {
    name   = "name"
    values = ["amzn2-ami-hvm-*-x86_64-gp2"]
  }
}

# Look up an existing VPC
data "aws_vpc" "existing" {
  tags = { Name = "main-vpc" }
}

# Use in a resource
resource "aws_instance" "app" {
  ami           = data.aws_ami.amazon_linux.id
  subnet_id     = data.aws_vpc.existing.id
  instance_type = "t3.micro"
}

Lifecycle & Dependencies

resource "aws_instance" "web" {
  ami           = data.aws_ami.latest.id
  instance_type = "t3.micro"

  # Lifecycle rules
  lifecycle {
    create_before_destroy = true    # Zero-downtime replacement
    prevent_destroy       = true    # Safety: block deletion
    ignore_changes        = [tags]  # Ignore external tag changes
  }

  # Explicit dependency (usually auto-detected)
  depends_on = [aws_iam_role_policy.s3_access]
}
  • Terraform auto-detects implicit dependencies from references
  • Use depends_on only when dependencies aren't visible in config
  • create_before_destroy is essential for zero-downtime updates

Expressions & Functions

# Conditional expression
instance_type = var.env == "prod" ? "t3.large" : "t3.micro"

# For expression - transform a list
upper_names = [for name in var.names : upper(name)]

# For expression - filter
prod_servers = [for s in var.servers : s if s.env == "prod"]

# Dynamic blocks
resource "aws_security_group" "web" {
  dynamic "ingress" {
    for_each = var.ingress_rules
    content {
      from_port   = ingress.value.port
      to_port     = ingress.value.port
      protocol    = "tcp"
      cidr_blocks = ingress.value.cidrs
    }
  }
}

Workspaces

  • Workspaces let you manage multiple environments from the same config
  • Each workspace has its own state file
  • Use terraform.workspace to reference the current workspace
# Create and switch workspaces
terraform workspace new staging
terraform workspace new production
terraform workspace select staging
terraform workspace list

# Use in config
resource "aws_instance" "app" {
  instance_type = terraform.workspace == "production" \
    ? "t3.large" : "t3.micro"

  tags = {
    Environment = terraform.workspace
  }
}

Importing Existing Resources

Bring manually created resources under Terraform management:

# Import block (Terraform 1.5+)
import {
  to = aws_instance.web
  id = "i-0abc123def456789"
}

# Generate config automatically
terraform plan -generate-config-out=generated.tf

# Or use CLI import (older approach)
terraform import aws_instance.web i-0abc123def456789

# Then write the matching resource block:
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"
  # ... match existing config
}

Best Practices

1 Always use remote state with locking for team collaboration
2 Run terraform plan and review before every apply
3 Use modules for reusable, DRY infrastructure code
4 Pin provider versions to avoid unexpected changes
5 Never commit .tfstate or .tfvars with secrets to git
6 Use variables for anything that differs between environments
7 Tag all resources with owner, environment, managed-by
8 Run terraform fmt and terraform validate in CI

Quick Start

# Install Terraform
brew install terraform          # macOS
# or download from terraform.io

# Create main.tf
cat <<EOF > main.tf
provider "aws" {
  region = "us-east-1"
}

resource "aws_s3_bucket" "demo" {
  bucket = "my-terraform-demo-bucket"
  tags   = { ManagedBy = "terraform" }
}

output "bucket_arn" {
  value = aws_s3_bucket.demo.arn
}
EOF

# Run it!
terraform init      # Download providers
terraform plan      # Preview changes
terraform apply     # Create resources
terraform destroy   # Clean up
