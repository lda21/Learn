Command Overview

What Are Slash Commands?

  • Built-in commands prefixed with /
  • Type /help to see all available commands
  • Tab-completion supported — type / then Tab
  • Some accept arguments: /model sonnet
  • Many have aliases: /quit = /exit

Command Categories

  • Session — clear, compact, resume, branch
  • Model & Performance — model, fast, effort, cost
  • Configuration — config, permissions, theme
  • Project & Memory — init, memory, hooks
  • Git & Code — diff, pr-comments, plan
  • Account & Auth — login, status, doctor
  • Integrations — ide, mcp, chrome
  • Skills — extensible slash commands

Built-in vs Marketplace Skills

Not all /commands ship with Claude Code. Here's how to tell the difference:

TypeSourceHow to GetExamples
Built-inShips with Claude CodeNothing — available immediately/help, /clear, /model, /compact, /diff, /plan, /config, /init
Bundled SkillPre-installed with Claude CodeAvailable out of the box/simplify, /batch, /debug, /loop
Marketplace SkillEverything Claude Code (ECC) marketplace or custom install/configure-ecc or manual install in ~/.claude/skills//tdd, /code-review, /python-review, /devfleet
PluginClaude Code plugin registry/plugin install <name>code-review@claude-plugins-official
MCP PromptConnected MCP serversConfigure via /mcpDynamic — depends on connected servers

Slides 1–11 = all built-in. Slides 13–19 are mostly marketplace skills that require installation via ECC or manual setup.

Development Workflow Skills

Marketplace / ECC — install via /configure-ecc or add to ~/.claude/skills/

/plan

Create a step-by-step implementation plan. Restates requirements, assesses risks, and waits for approval before coding.

/tdd

Test-driven development enforcement. Scaffold interfaces, generate tests FIRST, then implement to pass them.

/code-review

Comprehensive code review. Checks for bugs, security, performance, style, and maintainability issues.

/build-fix

Build the project and iteratively fix compilation errors until the build succeeds.

/verify

Comprehensive verification: lint, type-check, test, and build. Ensures everything passes before committing.

/refactor-clean

Refactor and clean unused code. Removes dead imports, unused functions, and simplifies logic.

/simplify bundled

Review changed code for quality and efficiency. Suggests cleaner, more reusable alternatives.

/e2e

Generate and run Playwright E2E tests. Creates test journeys, runs them, captures screenshots on failure.

How Reviews Work

  • Analyzes changed files in your branch
  • Checks against language best practices
  • Flags CRITICAL, HIGH, MEDIUM issues
  • Suggests specific fixes with code examples

Example Usage

# Review all changed Python files
/python-review

# Review after making changes
# Claude analyzes the diff and reports:
# - CRITICAL: SQL injection in query()
# - HIGH: Missing type hints on public API
# - MEDIUM: Consider using dataclass

How Build Fix Works

1. Runs the build command
2. Parses error output
3. Fixes errors incrementally
4. Re-runs until clean build

Example: /rust-build

Runs cargo build, interprets borrow checker errors, applies ownership fixes, resolves lifetime annotations, and retries until compilation succeeds.

TDD Workflow

  • RED — Write failing test first
  • GREEN — Write minimal code to pass
  • REFACTOR — Clean up, verify coverage
  • Target: 80%+ line coverage

Example: /go-test

# Claude generates table-driven tests:
func TestAdd(t *testing.T) {
  tests := []struct{
    a, b, want int
  }{
    {1, 2, 3},
    {0, 0, 0},
    {-1, 1, 0},
  }
  for _, tt := range tests {
    got := Add(tt.a, tt.b)
    assert.Equal(t, tt.want, got)
  }
}
