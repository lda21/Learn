What is Claude Code?

Overview

  • Agentic coding tool by Anthropic
  • Runs in your terminal alongside your editor
  • Reads, writes, and edits files directly
  • Runs commands, tests, and git operations
  • Understands entire codebases

How It Works

You describe the task
↓
Claude reasons & uses tools
↓
Files edited, tests run, code shipped

Installation & Setup

# Install globally via npm
npm install -g @anthropic-ai/claude-code

# Or use without installing
npx @anthropic-ai/claude-code

# Launch in your project directory
cd my-project
claude

# First run: authenticate with Anthropic API key
# or connect to Claude Pro/Max subscription

The Tool System

Read

Read any file. Supports images, PDFs, notebooks. Line ranges for large files.

Edit

Precise string replacements. Shows diffs. Preserves formatting and indentation.

Write

Create new files or complete rewrites. Use Edit for modifications instead.

Bash

Execute shell commands. Run tests, builds, git operations, system commands.

Glob

Find files by pattern. **/*.tsx, src/**/*.test.js

Grep

Search file contents with regex. Faster than shell grep. Multiple output modes.

Claude chooses tools autonomously based on the task. You approve each action.

Permission Modes

Control what Claude can do without asking. Three built-in modes:

Ask (Default)

Prompts for approval on every tool call. Safest mode for learning.

Auto-edit

Auto-approves Read, Edit, Write, Glob, Grep. Asks for Bash commands.

Full Auto

Auto-approves everything. Use in trusted environments or CI/CD.

Custom Rules

Fine-grained control in .claude/settings.json with allowedTools patterns.

// .claude/settings.json
{
  "permissions": {
    "allow": ["Read", "Glob", "Grep", "Edit", "Write"],
    "deny": ["Bash(rm *)"]
  }
}

CLAUDE.md Files

Project-level context that Claude reads automatically. Like a README for your AI assistant.

# CLAUDE.md

## Project Overview
This is a Next.js 14 app with App Router, TypeScript, and Prisma.

## Tech Stack
- Next.js 14 (App Router)
- TypeScript (strict mode)
- Prisma + PostgreSQL
- Tailwind CSS
- Jest + React Testing Library

## Conventions
- Use server components by default
- Prefer named exports
- Run `npm test` before committing
- Database migrations: `npx prisma migrate dev`

## Important Files
- src/lib/auth.ts - authentication logic
- src/app/api/ - API routes
- prisma/schema.prisma - database schema

Tip: Place in project root. Can also use per-directory CLAUDE.md for specific areas.

Hooks System

Run custom commands before or after Claude's tool calls.

// .claude/settings.json
{
  "hooks": {
    "preToolCall": [
      {
        "matcher": "Edit",
        "command": "echo 'File being edited: $CLAUDE_FILE_PATH'"
      }
    ],
    "postToolCall": [
      {
        "matcher": "Bash",
        "command": "echo 'Command completed'"
      }
    ],
    "onPromptSubmit": [
      {
        "command": "echo 'New prompt submitted'"
      }
    ]
  }
}

Use hooks for: linting after edits, notifications, audit logging, custom validation.

Sub-Agents & Parallelism

Claude spawns specialized sub-agents for complex tasks, running them in parallel.

Claude Agent SDK

Build custom AI agents powered by Claude. The SDK provides tool execution, conversation management, and agent orchestration.

import { Agent, tool } from "@anthropic-ai/agent-sdk";

const agent = new Agent({
  model: "claude-sonnet-4-6",
  tools: [
    tool("search_docs", {
      description: "Search internal documentation",
      parameters: { query: { type: "string" } },
      execute: async ({ query }) => searchDocs(query),
    }),
  ],
  system: "You are a helpful documentation assistant.",
});

const result = await agent.run("How do I configure SSO?");

Headless Mode & CI/CD

Run Claude Code non-interactively in CI pipelines, scripts, and automation workflows.

# Headless mode: pass prompt via -p flag
claude -p "fix all TypeScript errors in src/"

# Pipe input
echo "explain this error: $(cat error.log)" | claude -p

# Use in GitHub Actions
- name: Fix lint errors
  run: claude -p "fix all ESLint errors and commit"
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

# JSON output for scripting
claude -p "list all TODO comments" --output-format json

# Full auto mode for CI (no prompts)
claude -p "run tests and fix failures" --permission-mode full-auto

Use cases: Automated code fixes, PR review bots, migration scripts, release automation.

Best Practices

1. Always have a CLAUDE.md in your project root
2. Start with Ask mode, graduate to Auto-edit
3. Review diffs before accepting changes
4. Commit frequently — easy to rollback
5. Use /compact in long sessions
6. Break complex tasks into smaller requests
7. Paste full error messages for debugging
8. Set up MCP servers for your workflow
9. Use hooks for linting and validation
10. Trust but verify — always run tests
