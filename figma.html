What is Figma?

Overview

  • Browser-based design tool (also desktop app)
  • Real-time multiplayer collaboration
  • Design, prototype, and handoff in one tool
  • Free tier for up to 3 projects
  • Acquired by Adobe (2023, later cancelled)

Why Figma?

  • No install needed — works in any browser
  • Real-time collaboration like Google Docs
  • Single source of truth for design
  • Developer handoff built in (Dev Mode)
  • Massive plugin ecosystem
  • Free for individuals and education

Frames & Layout

Frames vs Groups

  • Frame — like an HTML div. Has size, clip, auto layout, constraints. Use for everything.
  • Group — just a container. No layout control. Avoid for complex designs.
  • Press F for frame tool
  • Device presets: iPhone, Desktop, Tablet

Constraints

  • Pin elements to frame edges
  • Left, Right, Center (horizontal)
  • Top, Bottom, Center (vertical)
  • Left & Right = stretch horizontally
  • Scale = resize proportionally
  • Essential for responsive frame resizing

Auto Layout

Figma's most powerful feature — think CSS Flexbox for design.

Direction

Horizontal (row) or Vertical (column). Wrap option for multi-line layouts.

Padding & Gap

Independent padding per side. Gap = space between children. Like CSS gap property.

Sizing

Hug = shrink to fit content. Fill = expand to parent. Fixed = exact size.

Alignment

9-point alignment grid (top-left, center, bottom-right, etc). Space between option.

Shortcut: Shift+A to add auto layout to any frame. Nest auto layouts for complex layouts.

Pro tip: Build buttons as auto layout frames: Icon + Text with padding = perfectly responsive button.

Components & Variants

Components

  • Reusable design elements (like React components)
  • Main component = source of truth
  • Instance = copy that inherits changes
  • Ctrl+Alt+K to create component
  • Override text, colors, visibility per instance
  • Detach instance to break link

Variants

  • Multiple states in one component
  • Properties: Size (sm, md, lg), State (default, hover, active), Type (primary, secondary)
  • Boolean props: show/hide icon
  • Instance swap: change nested component
  • Text props: expose editable text

Example: Button component with variants for size, style, state, and icon = one component replaces dozens of designs.

Design Tokens & Styles

Color Styles

Define brand colors: Primary/500, Neutral/100-900, Error, Success. Apply to fills and strokes.

Text Styles

Heading 1-6, Body, Caption, Overline. Font family, size, weight, line height, letter spacing.

Effect Styles

Drop shadows, inner shadows, blur. Elevation levels: shadow-sm, shadow-md, shadow-lg.

Variables (New)

Design tokens with modes (light/dark). Color, number, string, boolean. Alias and reference other variables.

Shared Libraries: Publish styles and components to your team. Everyone uses the same source of truth.

Typography & Colors

Typography Best Practices

  • Use a type scale (1.25 or 1.333 ratio)
  • Max 2 font families (heading + body)
  • Line height: 1.5 for body, 1.2 for headings
  • Popular: Inter, Plus Jakarta Sans, DM Sans
  • Use text styles for consistency

Color System

  • Primary — brand color (buttons, links)
  • Neutral — grays for text, borders, bg
  • Semantic — success, warning, error, info
  • Generate scales: 50, 100, 200...900
  • Check contrast: WCAG AA = 4.5:1 minimum
  • Use Variables for light/dark mode

Prototyping

Create interactive prototypes without code. Switch to the Prototype tab.

Interactions

Connect frames with wires. Triggers: click, hover, drag, keyboard, delay, mouse enter/leave.

Animations

Smart Animate detects matching layers and animates differences. Dissolve, move in, push, slide.

Overlays

Open frames as modals, dropdowns, tooltips. Position: centered, top-left, manual. Dismiss on click outside.

Scroll & Overflow

Vertical/horizontal scrolling. Fixed elements (headers, nav bars) stay in place. Clip content option.

Present: Press play button or Ctrl+Alt+Enter to preview. Share prototype link with stakeholders.

Developer Handoff

Inspect Panel

  • Select any element to see CSS properties
  • Spacing, sizing, colors, fonts, effects
  • Copy CSS, iOS, or Android code
  • Measure distances between elements
  • Export assets (SVG, PNG, PDF)

Dev Mode (Paid)

  • Dedicated developer view
  • Auto-generated code snippets
  • Compare design changes over time
  • Link to Jira, GitHub, VS Code
  • Section-ready annotations
  • Variable/token mapping

Export: Select element → bottom-right Export section → choose format (SVG for icons, PNG for images, PDF for docs).

Useful Plugins

Unsplash

Free stock photos directly in Figma. Fill shapes with beautiful images instantly.

Iconify

200,000+ icons from 100+ sets (Material, Heroicons, Lucide). Search and insert as vectors.

Content Reel

Realistic placeholder content: names, avatars, addresses, dates. Great for mockups.

Stark

Accessibility checker: contrast ratios, color blindness simulation, WCAG compliance.

Install plugins: Figma menu → Plugins → Browse. Run with Ctrl+/ then type plugin name.

Version History & Branching

Version History

  • Auto-saved every 30 minutes
  • Name versions manually for milestones
  • Browse and restore any version
  • Ctrl+Alt+S to save named version
  • Compare side-by-side with current

Branching (Paid)

  • Like Git branches for design files
  • Experiment without affecting main file
  • Merge branch back with conflict resolution
  • Review changes before merging
  • Great for design explorations

Naming convention: Use dates or feature names: "v2.1 - Dashboard Redesign - 2024-03-15"

Responsive Design

Techniques

  • Constraints — pin to edges, stretch
  • Auto Layout — Hug/Fill sizing
  • Min/Max width — set size boundaries
  • Text truncation — auto-truncate with ellipsis
  • Wrap — auto layout wrapping for grids

Breakpoint Strategy

  • Mobile: 375px (iPhone)
  • Tablet: 768px (iPad)
  • Desktop: 1440px
  • Design mobile-first, then expand
  • Use component variants per breakpoint

Tip: Create a responsive component that uses Auto Layout + Fill sizing. Resize the parent frame and watch children adapt.

Figma to Code

Built-in Export

  • CSS code in Inspect panel
  • SVG code for vector graphics
  • Export assets at 1x, 2x, 3x
  • PDF export for documentation

Code Generation Plugins

  • Locofy — Figma to React/Next.js
  • Anima — export to React, Vue, HTML
  • Figma Tokens — export design tokens as JSON
  • Figma API — programmatic access to files

Best practice: Use Figma for design intent, not pixel-perfect code generation. Build your own component library that maps to Figma components.

Best Practices

1. Use frames for everything (not groups)
2. Auto Layout on every frame
3. Name all layers meaningfully
4. Use styles/variables for all colors and text
5. Build components for repeated elements
6. Use variants instead of duplicate components
7. Organize pages: Cover, Components, Screens, Archive
8. Publish and version your design system
9. Use consistent spacing scale (multiples of 4 or 8)
10. Check accessibility (contrast, font size, touch targets)
