What Is Claw Code?

Origin Story

  • Clean-room reimplementation of Claude Code's agent harness
  • Created after TS source exposure on March 31, 2026
  • Captures architectural patterns without copying proprietary code
  • Built using oh-my-codex (OmX) orchestration
  • Repository: instructkr/claw-code

Key Numbers

  • 207 command entries mirrored
  • 184 tool entries mirrored
  • ~20K lines of Rust across 6 crates
  • 100+ Python source files
  • Original TS: 1,902 files across 35+ subsystems
  • 32 test cases in the verification suite

Why Study This Repo?

  • Agent Harness Engineering — Learn how production AI agent loops are built from scratch
  • Dual-Stack Architecture — Compare Python (porting fidelity) vs Rust (production performance) approaches
  • Permission Systems — Three-tier security model gating every tool execution
  • MCP Protocol — Model Context Protocol client implementation across 6 transport types
  • Streaming APIs — Server-Sent Events parsing for real-time AI responses
  • Session Management — Persistence, compaction, replay, and resume patterns
  • Config Hierarchies — Multi-layer configuration resolution (user → project → local)
  • CLI Design — Both Python argparse and Rust Clap approaches to rich CLI interfaces
  • Trait-based Abstraction — Rust generics over ApiClient and ToolExecutor traits
  • Parity Auditing — Systematic comparison against a reference codebase

class PortRuntime:
    """Routes prompts to matching commands/tools via token scoring."""

    def route_prompt(self, prompt: str) -> list[RoutedMatch]:
        """Tokenize prompt, score against all registries."""
        tokens = prompt.lower().split()
        matches = []
        for cmd in PORTED_COMMANDS:
            score = self._score(tokens, cmd.name)
            if score > 0:
                matches.append(RoutedMatch('command', cmd.name,
                                           cmd.source_hint, score))
        for tool in PORTED_TOOLS:
            score = self._score(tokens, tool.name)
            if score > 0:
                matches.append(RoutedMatch('tool', tool.name,
                                           tool.source_hint, score))
        return sorted(matches, key=lambda m: m.score, reverse=True)

permissions.py — Access Control

ToolPermissionContext


@dataclass(frozen=True)
class ToolPermissionContext:
    deny_names: frozenset  # Exact names
    deny_prefixes: tuple   # Name prefixes

    @classmethod
    def from_iterables(cls, names, prefixes):
        return cls(
            frozenset(n.lower() for n in names),
            tuple(p.lower() for p in prefixes)
        )

    def blocks(self, tool_name: str) -> bool:
        """Check if tool is denied."""
        lower = tool_name.lower()
        if lower in self.deny_names:
            return True
        return any(
            lower.startswith(p)
            for p in self.deny_prefixes
        )

How It Works

  • Exact name matching — deny specific tools
  • Prefix matching — deny tool families
  • All comparisons case-insensitive
  • Uses frozenset for immutable deny lists
  • Integrated into filter_tools_by_permission_context()
  • Denials tracked as PermissionDenial objects

transcript.py


class TranscriptStore:
    entries: list     # All messages
    flushed: bool     # Whether dumped to disk

    def append(self, entry):
        """Add a message to history."""
        self.entries.append(entry)

    def compact(self, keep_last=10):
        """Keep only last N entries.
        Reduces context window usage."""
        self.entries = self.entries[-keep_last:]

    def replay(self) -> list:
        """Return all entries for re-submission."""
        return list(self.entries)

    def flush(self):
        """Mark as flushed to disk."""
        self.flushed = True

Compaction is key — prevents context window overflow in long sessions

        match input.trim() {
            "/help"    => Some(Self::Help),
            "/status"  => Some(Self::Status),
            "/compact" => Some(Self::Compact),
            other      => Some(Self::Unknown(other.into())),
        }
    }
}

    TextDelta(String),               // Streaming text chunk
    ToolUse { id, name, input },     // Tool call request
    Usage(TokenUsage),               // Token accounting
    MessageStop,                     // End of response
}

The Turn Loop — Step by Step

1. User Input → Add to session messages
↓
2. Build ApiRequest (system_prompt + messages)
↓
3. Stream response via ApiClient.stream()
↓
4. Parse AssistantEvents (TextDelta, ToolUse, Usage)
↓
5. For each ToolUse: check PermissionPolicy
↓
6. Execute tool via ToolExecutor.execute()
↓
7. Append tool results to messages
↓
8. Loop until MessageStop OR max_iterations (50)

Auto-compaction triggers when input tokens exceed 200K — configurable via CLAUDE_CODE_AUTO_COMPACT_INPUT_TOKENS

fn parse_frame(line: &str) -> SseFrame {
    if line.starts_with("event: ") {
        SseFrame::Event(line[7..].into())
    } else if line.starts_with("data: ") {
        SseFrame::Data(line[6..].into())
    } else if line.is_empty() {
        SseFrame::EndOfEvent
    } else {
        SseFrame::Comment
    }
}

Key Design Patterns

Registry Pattern

Commands and tools loaded from JSON snapshots into typed registries with lookup, search, and filtering.

Trait-based Generics

ConversationRuntime<C, T> parameterized over ApiClient and ToolExecutor for testability.

Immutable Dataclasses

Python uses @dataclass(frozen=True) everywhere; Rust enforces ownership semantics.

Permission-First

Every tool execution gated by three-tier model. No tool runs without permission check.

Session-Centric State

All conversation state organized around a persistent session with compaction and resume.

Hook Pipeline

Pre/Post tool use hooks for extensibility — validation, mutation, logging, and denial.

Config Layering

User → Project → Local precedence with merged output for maximum flexibility.

Parity Auditing

Systematic comparison against reference implementation with coverage metrics.
