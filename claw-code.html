What Is Claw Code?

Origin Story

  • Clean-room reimplementation of Claude Code's agent harness in Python
  • Captures architectural patterns without copying proprietary code

  • Repository: instructkr/claw-code

Agent Loop Patterns

The Agent Turn Loop

1. Route prompt → tokenize & score against registries
↓
2. Check permissions via ToolPermissionContext
↓
3. Execute matched commands & tools
↓
4. Track usage (input/output tokens)
↓
5. Append to transcript & stream events
↓
6. Check compaction threshold
↓
7. Return TurnResult (output, stop_reason, usage)

Loop continues until stop_reason != 'completed' or max_turns reached

PortRuntime — Prompt Routing

# runtime.py - Actual routing implementation

class PortRuntime:
    def route_prompt(self, prompt: str, limit: int = 5) -> list[RoutedMatch]:
        tokens = {
            token.lower()
            for token in prompt.replace('/', ' ').replace('-', ' ').split()
            if token
        }
        by_kind = {
            'command': self._collect_matches(tokens, PORTED_COMMANDS, 'command'),
            'tool':    self._collect_matches(tokens, PORTED_TOOLS,    'tool'),
        }

        # Guarantee at least one of each kind (if available)
        selected: list[RoutedMatch] = []
        for kind in ('command', 'tool'):
            if by_kind[kind]:
                selected.append(by_kind[kind].pop(0))

        # Fill remaining slots from leftovers sorted by score
        leftovers = sorted(
            [m for matches in by_kind.values() for m in matches],
            key=lambda item: (-item.score, item.kind, item.name),
        )
        selected.extend(leftovers[: max(0, limit - len(selected))])
        return selected[:limit]

Permission System

ToolPermissionContext

# permissions.py
@dataclass(frozen=True)
class ToolPermissionContext:
    deny_names: frozenset[str] = \
        field(default_factory=frozenset)
    deny_prefixes: tuple[str, ...] = ()

    @classmethod
    def from_iterables(cls,
        deny_names=None,
        deny_prefixes=None):
        return cls(
            deny_names=frozenset(
                n.lower()
                for n in (deny_names or [])),
            deny_prefixes=tuple(
                p.lower()
                for p in (deny_prefixes or [])),
        )

    def blocks(self, tool_name: str) -> bool:
        lowered = tool_name.lower()
        return (
            lowered in self.deny_names
            or any(lowered.startswith(p)
                   for p in self.deny_prefixes)
        )

Session Store & Persistence

session_store.py

@dataclass(frozen=True)
class StoredSession:
    session_id: str
    messages: tuple[str, ...]
    input_tokens: int
    output_tokens: int

DEFAULT_SESSION_DIR = Path('.port_sessions')

def save_session(session, directory=None):
    target_dir = directory or DEFAULT_SESSION_DIR
    target_dir.mkdir(parents=True, exist_ok=True)
    path = target_dir / f'{session.session_id}.json'
    path.write_text(
        json.dumps(asdict(session), indent=2)
    )
    return path

def load_session(session_id, directory=None):
    target_dir = directory or DEFAULT_SESSION_DIR
    data = json.loads(
        (target_dir / f'{session_id}.json')
            .read_text()
    )
    return StoredSession(
        session_id=data['session_id'],
        messages=tuple(data['messages']),
        input_tokens=data['input_tokens'],
        output_tokens=data['output_tokens'],
    )

Transcript & Compaction

TranscriptStore

class TranscriptStore:
    entries: list
    flushed: bool = False

    def append(self, entry: str):
        """Add a message to history."""
        self.entries.append(entry)

    def compact(self, keep_last: int = 10):
        """Keep only last N entries.
        Reduces context window usage."""
        if len(self.entries) > keep_last:
            self.entries = \
                self.entries[-keep_last:]

    def replay(self) -> list[str]:
        """Return all for re-submission."""
        return list(self.entries)

    def flush(self):
        """Mark as flushed to disk."""
        self.flushed = True

Why Compaction Matters

Turn 1: 500 tokens
Turn 5: 2,500 tokens
Turn 12: 6,000 tokens
compact_after_turns = 12 triggers!
After compaction: ~1,500 tokens

Without compaction, token growth is linear per turn. Long sessions would exhaust the context window. Compaction keeps only the most recent entries.

Key Design Patterns

Immutability Everywhere

@dataclass(frozen=True) on every model. UsageSummary.add_turn() returns a new instance instead of mutating.

Registry Pattern

Commands and tools loaded from JSON snapshots via @lru_cache into typed tuple[PortingModule, ...] registries.

Permission-First

Every tool filtered through ToolPermissionContext before execution. No tool runs without a permission check.

Session-Centric State

All conversation state organized around a persistent StoredSession with compaction and resume.

Config Layering

User → Project → Local precedence with merged output. More specific overrides less specific.

Zero Dependencies

Entire Python stack uses only stdlib: json, dataclasses, pathlib, argparse, functools. No pip install needed.
