What Are Agents?

Definition

  • Markdown files with YAML frontmatter
  • No code required — just text
  • Placed in ~/.claude/agents/
  • Main session delegates tasks to them
  • Each gets own context window
  • Restricted tools & focused instructions

Agent Anatomy

# ~/.claude/agents/code-reviewer.md
---
name: code-reviewer
description: Reviews code for quality, security, and best practices
tools: ["Read", "Grep", "Glob"]        # read-only!
model: sonnet                           # fast execution
color: orange
---

You are an expert code reviewer.

## Your Role
- Review code changes for bugs, security issues, and style
- Provide severity ratings: CRITICAL / HIGH / MEDIUM / LOW
- Focus on what matters, skip nitpicks

## Workflow
1. Read the changed files
2. Check for security vulnerabilities (OWASP Top 10)
3. Check for logic errors and edge cases
4. Review error handling and logging

## Output Format
### [SEVERITY] filename:line — Issue title
Description of the issue and suggested fix.

Tool Restrictions & Model Routing

Tools = Safety Boundaries

# Read-only agent (reviewer)
tools: ["Read", "Grep", "Glob"]

# Implementation agent (builder)
tools: ["Read", "Grep", "Glob",
        "Bash", "Edit", "Write"]

# Research agent (no file changes)
tools: ["Read", "Grep", "Glob",
        "WebFetch", "WebSearch"]

Restrict tools to the minimum needed. A reviewer should never edit files.

Key Hooks

  • block-no-verify — prevents --no-verify
  • config-protection — blocks linter config edits

MCP Servers & Integration

What Are MCP Servers?

  • Model Context Protocol — standard for tool integration
  • External tools — databases, APIs, browsers, file systems
  • Dynamic loading — add capabilities without code changes
  • Context7 — fetch live docs for any library
  • Exa / Firecrawl — web search and scraping

MCPs turn agents into extensible platforms — plug in any external service.

Configuration

// .claude/settings.json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": [
        "-y",
        "@upstash/context7-mcp"
      ]
    },
    "postgres": {
      "command": "mcp-server-postgres",
      "args": ["postgresql://..."],
      "env": { "DB_SSL": "true" }
    }
  }
}

Each MCP server exposes tools that agents can call.

Build Your Own Agent

# Step 1: Create the file
touch ~/.claude/agents/my-agent.md

# Step 2: Define frontmatter
---
name: api-designer
description: Designs REST/gRPC APIs from requirements
tools: ["Read", "Grep", "Glob", "Write"]
model: sonnet
---

# Step 3: Write the system prompt
You are an API design expert.

## Workflow
1. Read the requirements document
2. Identify resources and relationships
3. Design endpoints with proper HTTP methods
4. Define request/response schemas
5. Write OpenAPI spec or .proto file

## Rules
- Use plural nouns for resources (/users not /user)
- Version APIs in URL (/v1/users)
- Use proper HTTP status codes
- Include pagination for list endpoints

Tips: Keep focused (one job), restrict tools to minimum, use opus only for deep reasoning.
