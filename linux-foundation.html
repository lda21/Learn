What is the Linux Foundation?

The Organization

  • Founded in 2000 as a nonprofit technology consortium
  • Vendor-neutral steward of open source collaboration
  • Hosts 900+ open source projects
  • 2M+ contributors worldwide
  • $16B+ in shared technology investment
  • Not a vendor — doesn’t sell commercial products

What It Provides

  • Governance — neutral project stewardship
  • Infrastructure — CI/CD, hosting, tooling (LFX)
  • Legal — IP management, license compliance
  • Marketing — events, branding, outreach
  • Training — courses, certifications, edX
  • Security — supply chain, auditing, best practices

Linux Kernel

The Flagship Project

  • Created by Linus Torvalds in 1991
  • 30M+ lines of code
  • 20,000+ contributors from 1,700+ companies
  • New release every ~9 weeks
  • Powers Android, cloud servers, supercomputers, embedded devices
  • 100% of Top 500 supercomputers run Linux
uname -r                  # kernel version
lsmod                     # loaded modules
modprobe <module>         # load a module
cat /proc/cpuinfo         # CPU info
dmesg | tail              # kernel messages

Key Subsystems

  • Process Scheduler — CFS (Completely Fair Scheduler)
  • Memory Management — virtual memory, page cache, OOM killer
  • VFS — Virtual Filesystem (ext4, XFS, Btrfs, ZFS)
  • Networking Stack — TCP/IP, eBPF, netfilter, XDP
  • Device Drivers — largest subsystem by code volume
  • Security — SELinux, AppArmor, seccomp, namespaces
  • cgroups — resource limits (foundation of containers)

Kubernetes (CNCF)

Container Orchestration

  • Originally from Google (Borg), donated to CNCF in 2015
  • Graduated CNCF project (2018)
  • 88K+ GitHub stars, 3,700+ contributors
  • Declarative: describe desired state, K8s reconciles
  • Self-healing, auto-scaling, rolling updates
  • De facto standard for container orchestration

Model Context Protocol (MCP)

What is MCP?

  • Open protocol for connecting AI models to external data and tools
  • Standardizes how LLMs interact with systems
  • Originated at Anthropic, donated to the Linux Foundation in 2025
  • "USB-C for AI" — one universal connector
  • Replaces fragile, custom integrations
  • Adopted by Claude, VS Code, JetBrains, Cursor, and more

PyTorch (LF AI & Data)

Deep Learning Framework

  • Created by Meta AI, moved to LF in 2022
  • Dynamic computation graphs — define-by-run
  • Pythonic, intuitive API beloved by researchers
  • Dominant in ML research papers (>80%)
  • Production-ready with TorchServe and ONNX export
  • 82K+ GitHub stars

Ecosystem

  • TorchServe — model serving at scale
  • TorchVision — computer vision
  • TorchAudio — audio processing
  • TorchText — NLP utilities
  • PyTorch Lightning — structured training
  • torch.compile — JIT compilation (2.0+)

OpenSearch

Search & Analytics

  • Community-driven fork of Elasticsearch + Kibana
  • Created by AWS in 2021, moved to LF in 2024
  • Apache 2.0 license — truly open source
  • Full-text search, log analytics, observability, security
  • Drop-in replacement for Elasticsearch 7.x
  • Vector search for AI/ML applications

Components

  • OpenSearch — search and analytics engine
  • OpenSearch Dashboards — visualization UI
  • Data Prepper — data ingestion pipeline
  • Clients — Java, Python, Go, Node.js, Ruby

RISC-V International

Open Instruction Set Architecture

  • Free, open ISA — no licensing fees
  • Started at UC Berkeley in 2010, joined LF in 2020
  • Modular design: base integer + optional extensions
  • Growing across embedded, mobile, automotive, HPC
  • 13B+ RISC-V cores shipped
  • Backed by Qualcomm, Samsung, NVIDIA, Google, Intel

OpenSSF — Open Source Security

Securing the Supply Chain

  • Founded 2020 after major supply chain attacks (SolarWinds, Log4Shell)
  • Cross-industry collaboration on open source security
  • Funded by Google, Microsoft, AWS, GitHub, Intel, and more
  • $150M+ pledged for supply chain security
SLSA LevelRequirements
Level 1Documented build process
Level 2Hosted build, signed provenance
Level 3Hardened builds, non-falsifiable provenance
Level 4Two-party review, hermetic builds

Key Initiatives

  • Scorecard — automated security health checks for repos (CI/CD, branch protection, fuzzing, SAST)
  • Sigstore — keyless code signing and verification (cosign, Rekor, Fulcio)
  • SLSA — Supply-chain Levels for Software Artifacts framework
  • GUAC — Graph for Understanding Artifact Composition
  • Alpha-Omega — funding security for critical projects
  • Package Analysis — detecting malicious packages

SPDX & Compliance Standards

SPDX — Software Bill of Materials

  • ISO/IEC 5962 international standard
  • Machine-readable format for SBOMs
  • Documents packages, files, licenses, relationships
  • Required by US Executive Order 14028 (cybersecurity)
  • Supported by GitHub, SPDX-SBOM-Generator, Syft
SPDXVersion: SPDX-2.3
DataLicense: CC0-1.0
SPDXID: SPDXRef-DOCUMENT
DocumentName: my-application
Creator: Tool: syft-0.98.0
PackageName: express
PackageVersion: 4.18.2
PackageLicenseConcluded: MIT

Compliance & Data Standards

  • OpenChain (ISO 5230) — international standard for open source license compliance programs
  • CDLA — Community Data License Agreement for sharing open data (Permissive + Sharing variants)
  • FOSSology — open source license compliance toolkit
  • REUSE — best practices for declaring licenses in repos
