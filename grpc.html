Why gRPC?

Overview

  • Created by Google, open-source (CNCF)
  • HTTP/2 — multiplexed, binary
  • Protobuf — binary serialization
  • Polyglot — 10+ language support
  • Contract-first API design
  • Bi-directional streaming built-in

Protocol Buffers (Protobuf)

syntax = "proto3";
package user.v1;

// Messages define data structures
message User {
  string id = 1;
  string name = 2;
  string email = 3;
  repeated string roles = 4;    // list of strings
  google.protobuf.Timestamp created_at = 5;
}

message GetUserRequest {
  string id = 1;
}

message ListUsersResponse {
  repeated User users = 1;
  int32 total_count = 2;
}

// Services define RPC methods
service UserService {
  rpc GetUser (GetUserRequest) returns (User);
  rpc CreateUser (User) returns (User);
  rpc ListUsers (ListUsersRequest) returns (ListUsersResponse);
  rpc WatchUsers (WatchRequest) returns (stream User);  // server streaming
}

4 Communication Patterns

1. Unary

Single request → single response. Like a normal function call.

rpc GetUser (Req) returns (User)

2. Server Streaming

Single request → stream of responses. Real-time feeds.

rpc Watch (Req) returns (stream Event)

3. Client Streaming

Stream of requests → single response. File uploads, batching.

rpc Upload (stream Chunk) returns (Status)

4. Bidirectional Streaming

Stream ↔ stream. Chat, gaming, collaborative editing.

rpc Chat (stream Msg) returns (stream Msg)

HTTP/2 Under the Hood

HTTP/2 (gRPC)

  • Multiplexing — many requests on one TCP connection
  • Binary framing — efficient parsing
  • Header compression — HPACK algorithm
  • Server push — proactive responses
  • Stream prioritization

HTTP/1.1 (REST)

  • One request per connection (or pipelining)
  • Text-based — larger payloads
  • No header compression
  • Head-of-line blocking
  • Need many TCP connections

Unary RPC Example

Server (Go)

func (s *server) GetUser(
  ctx context.Context,
  req *pb.GetUserRequest,
) (*pb.User, error) {

  user, err := s.db.FindByID(
    ctx, req.GetId(),
  )
  if err != nil {
    return nil, status.Error(
      codes.NotFound,
      "user not found",
    )
  }
  return user, nil
}

Server Streaming Example

// Proto definition
service StockService {
  rpc WatchPrice (StockRequest) returns (stream PriceUpdate);
}

// Server (Go) — sends stream of prices
func (s *server) WatchPrice(
  req *pb.StockRequest,
  stream pb.StockService_WatchPriceServer,
) error {
  ticker := time.NewTicker(time.Second)
  defer ticker.Stop()

  for {
    select {
    case <-stream.Context().Done():
      return nil  // client disconnected
    case <-ticker.C:
      price := getLatestPrice(req.Symbol)
      if err := stream.Send(&pb.PriceUpdate{
        Symbol: req.Symbol,
        Price:  price,
        Time:   timestamppb.Now(),
      }); err != nil {
        return err
      }
    }
  }
}

Client Streaming Example

Server (Go)

func (s *server) UploadMetrics(
  stream pb.MetricsService_UploadMetricsServer,
) error {

  var count int
  for {
    metric, err := stream.Recv()
    if err == io.EOF {
      // client done — send summary
      return stream.SendAndClose(
        &pb.UploadSummary{
          Received: int32(count),
        },
      )
    }
    if err != nil {
      return err
    }
    s.store.Save(metric)
    count++
  }
}

Client (Python)

def generate_metrics():
    """Stream batch of metrics."""
    for cpu, mem in collect_samples():
        yield pb.Metric(
            cpu_percent=cpu,

            timestamp=now(),

# Open stream, send all, get summary
summary = stub.UploadMetrics(

Client sends many → server replies once. Ideal for batch uploads, telemetry, file transfer.

Bidirectional Streaming

// Proto definition
service ChatService {
  rpc Chat (stream ChatMessage) returns (stream ChatMessage);
}

// Server — receives and broadcasts messages
func (s *server) Chat(
  stream pb.ChatService_ChatServer,
) error {
  for {
    msg, err := stream.Recv()
    if err == io.EOF {
      return nil
    }
    if err != nil {
      return err
    }
    // Broadcast to all connected clients
    for _, client := range s.clients {
      client.Send(&pb.ChatMessage{
        User:    msg.User,
        Text:    msg.Text,
        SentAt:  timestamppb.Now(),
      })
    }
  }
}

Metadata & Interceptors

Metadata (like HTTP headers)

// Client — send metadata
md := metadata.Pairs(
  "authorization", "Bearer "+token,
  "request-id", uuid.New().String(),
)
ctx := metadata.NewOutgoingContext(
  ctx, md,
)
resp, err := client.GetUser(ctx, req)

Interceptors (middleware)

// Server interceptor — auth + logging
func authInterceptor(
  ctx context.Context,
  req interface{},
  info *grpc.UnaryServerInfo,
  handler grpc.UnaryHandler,
) (interface{}, error) {
  md, _ := metadata.FromIncomingContext(ctx)
  token := md["authorization"][0]
  if !validateToken(token) {
    return nil, status.Error(
      codes.Unauthenticated, "bad token")
  }
  log.Printf("RPC: %s", info.FullMethod)
  return handler(ctx, req)  // proceed
}

Deadlines & Timeouts

Deadlines propagate across service calls — critical for microservice chains to prevent cascading hangs.

Setting Deadlines

// Client sets a 5-second deadline
ctx, cancel := context.WithTimeout(
  context.Background(),
  5*time.Second,
)
defer cancel()

resp, err := client.GetUser(ctx, req)
if status.Code(err) ==
   codes.DeadlineExceeded {
  log.Println("Timed out!")
}

Load Balancing

Client-Side LB

  • Client knows all server addresses
  • Round-robin, pick-first, weighted
  • Built into gRPC core
  • No extra infrastructure
  • Best for internal services
conn, _ := grpc.Dial(
  "dns:///my-service:50051",
  grpc.WithDefaultServiceConfig(
    `{"loadBalancingPolicy":"round_robin"}`),
)

Proxy-Based LB

  • Envoy — L7 gRPC-aware proxy
  • Istio — service mesh with gRPC support
  • Linkerd — lightweight service mesh
  • Health checking protocol built-in
  • Best for cross-team / external

gRPC-Web & gRPC Gateway

Browsers can't use gRPC directly (no HTTP/2 framing control). Two solutions:

gRPC-Web

  • Browser-compatible gRPC variant
  • Proxy translates HTTP/1.1 ↔ HTTP/2
  • Envoy or grpc-web proxy
  • Typed client from .proto

gRPC Gateway

  • Generates REST API from .proto
  • Annotate RPCs with HTTP mappings
  • Auto-generates OpenAPI/Swagger
  • REST clients + gRPC clients coexist
rpc GetUser (GetUserRequest) returns (User) {
  option (google.api.http) = {
    get: "/v1/users/{id}"
  };
}

Best Practices

1 Design APIs proto-first — schema is your contract
2 Set deadlines on every call — prevent cascading hangs
3 Handle all status codes — don't swallow errors
4 Use interceptors for auth, logging, tracing
5 Version with package names — user.v1, user.v2
6 Keep messages small — stream large data
7 Use grpcurl + reflection for debugging
8 Implement health checking protocol
