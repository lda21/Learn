TCP/IP Model

4-Layer Model

  • Application — HTTP, DNS, SSH, DHCP
  • Transport — TCP (reliable), UDP (fast)
  • Internet — IP, ICMP, ARP
  • Network Access — Ethernet, Wi-Fi

OSI vs TCP/IP

  • OSI = reference model (7 layers)
  • TCP/IP = practical model (4 layers)
  • OSI layers 5-7 → TCP/IP Application
  • OSI layers 1-2 → TCP/IP Network Access
  • TCP/IP is what the internet actually uses

IPv6 Fundamentals

Why IPv6?

  • IPv4 has ~4.3 billion addresses (exhausted)
  • IPv6 = 128-bit = 3.4 × 1038 addresses
  • No NAT needed — end-to-end connectivity
  • Built-in IPsec, simplified headers

2001:0db8:85a3::8a2e:0370:7334

Address Types

  • Unicast — one-to-one (Global, Link-local fe80::)
  • Multicast — one-to-many (ff00::/8)
  • Anycast — one-to-nearest (same as unicast format)
  • No broadcast in IPv6 (uses multicast instead)

MAC Addresses

  • 48-bit hardware address (Layer 2)
  • Format: AA:BB:CC:DD:EE:FF
  • First 3 bytes = OUI (vendor)
  • Last 3 bytes = unique identifier

Switch Operation

  • Learn — record source MAC + port
  • Forward — send to known destination port
  • Flood — unknown destination → all ports
  • Filter — don't forward to source port

VLANs

Virtual LANs segment a physical switch into isolated broadcast domains.

VLAN Basics

  • Each VLAN = separate broadcast domain
  • Traffic between VLANs requires a router
  • VLAN 1 = default (avoid using it)
  • Access ports carry one VLAN

802.1Q Trunking

  • Trunk ports carry multiple VLANs
  • Inserts 4-byte VLAN tag into frame
  • Native VLAN = untagged traffic
  • DTP negotiates trunks (disable in production)

Routing Fundamentals

Static vs Dynamic

  • Static — manually configured, no overhead
  • Dynamic — protocols discover routes automatically
  • Static for small/stub networks
  • Dynamic for large/changing topologies
Router(config)# ip route 10.1.0.0 255.255.0.0 192.168.1.1

OSPF

Open Shortest Path First — link-state protocol using Dijkstra's algorithm.

Areas

Hierarchical design. Area 0 = backbone. All areas connect to Area 0. Reduces LSA flooding.

DR / BDR

Designated Router elected on multi-access networks to reduce adjacencies. Highest priority or Router ID wins.

Cost Metric

Cost = Reference BW / Interface BW. Default ref = 100 Mbps. Lower cost = preferred path.

auto-cost reference-bandwidth 10000

Access Control Lists

Standard ACL (1-99)

  • Filters by source IP only
  • Place close to the destination
access-list 10 deny 10.1.1.0 0.0.0.255
access-list 10 permit any
!
interface gi0/1
 ip access-group 10 out

Extended ACL (100-199)

  • Filters by source, dest, protocol, port
  • Place close to the source
access-list 110 permit tcp 10.1.0.0 0.0.0.255 host 10.2.1.10 eq 443
access-list 110 deny ip any any
!
interface gi0/0
 ip access-group 110 in

Implicit deny any at the end of every ACL. Order matters — first match wins!

DHCP & DNS

DHCP (Dynamic Host Config)

Automatically assigns IP, mask, gateway, DNS.

DORA Process:

  • Discover — client broadcasts
  • Offer — server responds with IP
  • Request — client accepts offer
  • Ack — server confirms lease
ip dhcp pool OFFICE
 network 10.1.1.0 255.255.255.0
 default-router 10.1.1.1
 dns-server 8.8.8.8

DNS Resolution Flow

  • 1. Check local cache
  • 2. Query recursive resolver
  • 3. Root server → TLD server
  • 4. Authoritative server → IP
  • 5. Cache result (TTL)

Common records:

  • A — name → IPv4
  • AAAA — name → IPv6
  • CNAME — alias
  • MX — mail server

Network Security Basics

Port Security

Limit MAC addresses per port. Violation modes: protect, restrict, shutdown.

switchport port-security maximum 2

DHCP Snooping

Prevents rogue DHCP servers. Trusted ports (uplinks) vs untrusted ports (access).

ip dhcp snooping vlan 10

DAI (Dynamic ARP Inspection)

Validates ARP packets against DHCP snooping binding table. Prevents ARP spoofing.

ip arp inspection vlan 10

AAA Framework

Authentication (who?), Authorization (what?), Accounting (log it). Uses RADIUS or TACACS+.

Troubleshooting Methodology

  • 1. Define the problem
  • 2. Gather information
  • 3. Analyze information
  • 4. Test hypothesis
  • 5. Solve & document

Bottom-Up Approach

  • Physical: cable, link lights?
  • Data Link: MAC, VLAN, STP?
  • Network: IP, routing, ACL?
  • Transport: port open, firewall?
