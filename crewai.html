Crew AI

Multi-Agent AI Framework for Collaborative Intelligence

Use ← → arrow keys or buttons to navigate

What is Crew AI?

  • An open-source multi-agent orchestration framework built in Python
  • Enables role-based AI agents to collaborate on complex tasks
  • Think of it as assembling a team of specialized AI workers
  • Agents have roles, goals, backstories — like real team members
  • Supports OpenAI, Anthropic, local LLMs and 100+ tools

Key Features

Role-Based Agents

Each agent has a specific role, goal, and backstory that shapes its behavior

Task Delegation

Agents can delegate sub-tasks to other agents with the right expertise

Tool Integration

Agents use tools: web search, file I/O, APIs, code execution, and custom tools

Memory System

Short-term, long-term, and entity memory for context across interactions

Core Concept: Agents

An Agent is defined by its role, goal, and backstory:

from crewai import Agent

researcher = Agent(
    role="Senior Research Analyst",
    goal="Find and analyze cutting-edge AI developments",
    backstory="""You are an expert research analyst with 10 years
    of experience tracking AI trends. You're known for finding
    the most relevant and impactful information.""",
    tools=[search_tool, scrape_tool],
    llm="anthropic/claude-sonnet-4-6",
    verbose=True,
    allow_delegation=True,
)

Core Concept: Tasks

A Task is a specific assignment given to an agent:

from crewai import Task

research_task = Task(
    description="""Research the latest developments in AI agents
    for the year 2024. Focus on:
    1. Key frameworks and tools
    2. Notable real-world applications
    3. Emerging trends and predictions""",
    expected_output="""A detailed report with sections covering
    frameworks, applications, and trends. Include sources.""",
    agent=researcher,
    output_file="research_report.md",
)
  • description — what needs to be done (be specific!)
  • expected_output — what the result should look like
  • agent — who is responsible for this task

Core Concept: Crews

A Crew brings agents and tasks together:

from crewai import Crew, Process

crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[research_task, writing_task, editing_task],
    process=Process.sequential,   # or Process.hierarchical
    memory=True,
    verbose=True,
)

# Run the crew
result = crew.kickoff()
print(result)

Memory System

Short-Term Memory

Context within the current crew execution. Shared between agents during a run.

Long-Term Memory

Persists across executions. Agents learn and improve over time.

Entity Memory

Tracks information about specific entities (people, orgs, concepts).

Contextual Memory

Maintains conversation context and task relationships.

Full Example: Blog Writer Crew

from crewai import Agent, Task, Crew, Process

# Agents
researcher = Agent(
    role="Researcher", goal="Find accurate info",
    backstory="Expert researcher", tools=[search_tool])

writer = Agent(
    role="Writer", goal="Write engaging blog posts",
    backstory="Skilled content writer")

# Tasks
research = Task(
    description="Research {topic} thoroughly",
    expected_output="Key facts and insights",
    agent=researcher)

write = Task(
    description="Write a blog post using the research",
    expected_output="A 1000-word blog post in markdown",
    agent=writer, output_file="blog.md")

# Crew
crew = Crew(
    agents=[researcher, writer],
    tasks=[research, write],
    process=Process.sequential)

result = crew.kickoff(inputs={"topic": "AI Agents in 2024"})

Agent Delegation

  • Agents can delegate tasks to other agents when they lack expertise
  • Enable with allow_delegation=True on the agent
  • The delegating agent decides who should handle a sub-task
  • Works best with hierarchical process type
manager = Agent(
    role="Project Manager",
    goal="Coordinate the team to deliver results",
    backstory="Experienced PM who delegates effectively",
    allow_delegation=True,
)

crew = Crew(
    agents=[manager, researcher, writer, designer],
    tasks=[project_task],
    process=Process.hierarchical,
    manager_agent=manager,
)

Dynamic Inputs & Structured Output

from pydantic import BaseModel

# Structured output with Pydantic
class BlogPost(BaseModel):
    title: str
    content: str
    tags: list[str]

writing_task = Task(
    description="Write a blog about {topic}",
    expected_output="A complete blog post",
    agent=writer,
    output_pydantic=BlogPost,     # Structured output!
)

# Pass dynamic inputs at runtime
result = crew.kickoff(inputs={
    "topic": "Multi-Agent AI Systems"
})

# Access structured data
post = result.pydantic
print(post.title)
print(post.tags)

Best Practices

1 Write detailed backstories — they heavily influence agent behavior
2 Be specific in task descriptions and expected outputs
3 Use sequential for simple pipelines, hierarchical for complex ones
4 Give agents only the tools they need — less is more
5 Enable memory for crews that run repeatedly
6 Use output_pydantic for structured, predictable results
7 Test with verbose=True to debug agent reasoning
8 Start simple — 2-3 agents before scaling up

Quick Start

# Install Crew AI
pip install crewai crewai-tools

# Create a new project
crewai create crew my_crew
cd my_crew

# Project structure:
# my_crew/
#   src/my_crew/
#     config/
#       agents.yaml     # Agent definitions
#       tasks.yaml      # Task definitions
#     crew.py           # Crew assembly
#     main.py           # Entry point
#   pyproject.toml

# Set your API key
export OPENAI_API_KEY="sk-..."
# or
export ANTHROPIC_API_KEY="sk-ant-..."

# Run the crew
crewai run
