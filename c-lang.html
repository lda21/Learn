C# vs C Mental Model

C# — Managed World

  • Garbage collector frees memory for you
  • .NET runtime handles exceptions, bounds checks
  • JIT compiles IL to native at runtime
  • Rich standard library (LINQ, async, collections)
  • Type-safe references — no raw memory access

C — Bare Metal

  • You allocate and free every byte
  • No runtime — compiles directly to machine code
  • No exceptions, no bounds checks, no GC
  • Minimal standard library (stdio, stdlib, string)
  • Raw pointers — full control over memory layout

What you gain: Predictable performance, zero overhead, hardware control, understanding of how computers actually work.

What you lose: Safety nets, convenience, productivity. Every bug is your responsibility.

No Classes, No Objects

C# — Class with Methods

public class Player {
    public string Name { get; set; }
    public int Health { get; set; }

    public void TakeDamage(int amount) {
        Health -= amount;
        if (Health < 0) Health = 0;
    }
}

var p = new Player {
    Name = "Hero", Health = 100
};
p.TakeDamage(30);

C — Struct + Functions

typedef struct {
    char name[64];
    int health;
} Player;

void player_take_damage(Player *p,
                        int amount) {
    p->health -= amount;
    if (p->health < 0) p->health = 0;
}

Player p = {"Hero", 100};
player_take_damage(&p, 30);
  • No inheritance, no interfaces, no generics, no method overloading
  • Structs hold data only — functions are standalone and take a pointer to the struct
  • Naming convention: modulename_action() replaces object.Method()

Pointers — The Big One

In C#, every class variable is secretly a pointer. In C, you see and control them directly.

int x = 42;
int *p = &x;    // p holds the MEMORY ADDRESS of x

// &x  = "address of x"     (like getting a reference)
// *p  = "value at address"  (like dereferencing)
// p   = the address itself  (e.g., 0x7ffd5e8a3b2c)

*p = 100;       // x is now 100 (modified through the pointer)
printf("%d\n", x);   // prints 100
printf("%p\n", p);   // prints the memory address
x = 42
addr: 0x1000
←
p = 0x1000
points to x
// C# analogy:
// MyClass obj = new MyClass();  // obj IS a pointer (hidden)
// In C, you manage pointers explicitly:
Player *p = malloc(sizeof(Player));  // allocate on heap
p->health = 100;                    // -> dereferences AND accesses member
free(p);                             // YOU must free it

Pointer Arithmetic

In C#, you never do math on references. In C, pointer arithmetic is fundamental.

int arr[] = {10, 20, 30, 40, 50};
int *p = arr;        // arrays decay to pointers

// p + 1 does NOT add 1 byte!
// It adds sizeof(int) bytes (typically 4 bytes)
printf("%d\n", *(p + 0));  // 10  (arr[0])
printf("%d\n", *(p + 1));  // 20  (arr[1])
printf("%d\n", *(p + 3));  // 40  (arr[3])

// Array indexing IS pointer arithmetic:
// arr[3] is IDENTICAL to *(arr + 3)
// And even 3[arr] works! (because addition is commutative)

Arrays Are Not Objects

C# — Arrays are Objects

int[] arr = new int[5];
arr[0] = 42;

// Safe:
Console.WriteLine(arr.Length); // 5
arr[10] = 1; // IndexOutOfRange!

// Resizable:
var list = new List<int>();
list.Add(42);

C — Arrays are Raw Memory

int arr[5];           // stack array
arr[0] = 42;

// NO .Length, NO bounds checking:
arr[10] = 1; // compiles fine!
             // corrupts memory silently

// Heap array:
int *arr2 = malloc(5 * sizeof(int));
// Must track size yourself:
size_t len = 5;
free(arr2);
  • Arrays decay to pointers when passed to functions — size information is lost
  • You must always pass the length separately: void process(int *arr, size_t len)
  • Buffer overruns are the #1 source of security vulnerabilities in C code
  • No resizing — use realloc() for heap arrays (copies entire block)

Strings Are Char Arrays

C# — String Class

string name = "Alice";
string greeting = "Hello, " + name;
int len = name.Length;       // 5
bool eq = name == "Alice";   // true
string upper = name.ToUpper();
string sub = name.Substring(0, 3);
// Immutable, GC-managed, Unicode

C — Null-Terminated char*

char name[] = "Alice";
// In memory: {'A','l','i','c','e','\0'}
// That \0 is the null terminator!

size_t len = strlen(name);   // 5
char buf[64];
strcpy(buf, "Hello, ");      // DANGER!
strcat(buf, name);           // buffer overflow?

// Safe version:
snprintf(buf, sizeof(buf),
         "Hello, %s", name);

int eq = strcmp(name, "Alice"); // 0 = equal
  • strlen() walks memory until it finds \0 — O(n) every time (no cached .Length)
  • strcpy/strcat don't check buffer sizes — always prefer snprintf
  • Strings are mutable in C — you can modify characters in place
  • No Unicode support by default — char is a single byte (ASCII)

Memory Management

"In C#, the GC does this. In C, you ARE the GC."

#include <stdlib.h>

// malloc: allocate uninitialized memory
int *arr = malloc(10 * sizeof(int));  // 10 ints on heap
if (arr == NULL) { /* handle allocation failure! */ }

// calloc: allocate AND zero-initialize
int *arr2 = calloc(10, sizeof(int));  // all zeros

// realloc: resize existing allocation
arr = realloc(arr, 20 * sizeof(int)); // now 20 ints

// free: release memory back to OS
free(arr);
free(arr2);
arr = NULL;   // good practice: prevent use-after-free

Memory Leak

malloc without free. Memory grows until process dies.

int *p = malloc(100); /* forgot free(p) */

Double Free

Freeing the same pointer twice. Undefined behavior — crashes or worse.

free(p); free(p); /* BOOM */

Use-After-Free

Accessing memory after freeing it. Data may be overwritten by another allocation.

free(p); p->x = 5; /* reading garbage */

Dangling Pointer

Pointer to freed memory. Set to NULL after freeing.

free(p); p = NULL; /* safe */

Stack vs Heap

Stack (Automatic)

void foo() {
    int x = 42;        // stack
    char buf[256];      // stack
    Player p = {0};     // stack

    // All freed automatically
    // when foo() returns
}
// Fast allocation (just moves
// the stack pointer)
// Limited size (~1-8 MB)

Heap (Manual)

void bar() {
    int *x = malloc(sizeof(int));
    *x = 42;           // heap
    char *buf = malloc(256);
    Player *p = calloc(1,
                 sizeof(Player));

    // Must free manually:
    free(x); free(buf); free(p);
}
// Slower allocation (OS call)
// Virtually unlimited size
        
// CLASSIC BUG: returning pointer to stack variable
int *broken() {
    int x = 42;
    return &x;      // DANGLING POINTER! x is gone after return
}

// FIX: allocate on heap
int *correct() {
    int *x = malloc(sizeof(int));
    *x = 42;
    return x;       // caller must free(x) later
}

// In C#, the GC would keep the object alive. In C, the stack
// frame is destroyed the moment the function returns.

Structs & Enums

C# — Class with Methods

public class Vector2 {
    public float X { get; set; }
    public float Y { get; set; }

    public float Length() =>
        MathF.Sqrt(X*X + Y*Y);
}

enum Color { Red, Green, Blue }

C — Struct + Functions

typedef struct {
    float x;
    float y;
} Vector2;

float vec2_length(const Vector2 *v) {
    return sqrtf(v->x*v->x
               + v->y*v->y);
}

typedef enum {
    COLOR_RED, COLOR_GREEN, COLOR_BLUE
} Color;
        
// Struct padding & alignment: the compiler inserts gaps!
typedef struct {
    char  a;    // 1 byte
                // 3 bytes padding (to align int)
    int   b;    // 4 bytes
    char  c;    // 1 byte
                // 3 bytes padding (to align struct size)
} Padded;       // sizeof = 12, not 6!

// Use sizeof() to check. Never assume struct size.
// Pass large structs by pointer, not by value:
void process(const Player *p);  // good: passes 8-byte pointer
void process(Player p);         // bad: copies entire struct

The Preprocessor

C has a text substitution engine that runs before the compiler sees your code. C# has nothing like this.

// Simple constants (prefer over magic numbers)
#define MAX_PLAYERS 64
#define PI 3.14159265358979

// Macros with arguments (inline text replacement)
#define MAX(a, b)  ((a) > (b) ? (a) : (b))
#define SQUARE(x)  ((x) * (x))

// DANGER: macros are text substitution, not functions!
int val = SQUARE(i++);  // expands to ((i++) * (i++))
                        // i gets incremented TWICE!

// Conditional compilation
#ifdef _WIN32
    #include <windows.h>
#elif defined(__linux__)
    #include <unistd.h>
#endif

// Debug logging (commonly used pattern)
#ifdef DEBUG
    #define LOG(fmt, ...) fprintf(stderr, fmt, ##__VA_ARGS__)
#else
    #define LOG(fmt, ...) ((void)0)
#endif

// Stringify and token pasting
#define STR(x)    #x          // STR(hello) -> "hello"
#define CONCAT(a,b) a##b     // CONCAT(foo,bar) -> foobar

Rule of thumb: Prefer static inline functions over macros. Use macros only for conditional compilation and constants.

Function Pointers

C# — Delegates / Func<>

// Delegate type
Func<int, int, int> compare;
compare = (a, b) => a - b;

// Usage
var sorted = list.OrderBy(x => x.Name);

// Events
button.Click += OnClick;

C — Function Pointers

// Declaration:
// return_type (*name)(param_types)
int (*compare)(const void *,
               const void *);

int cmp_int(const void *a,
            const void *b) {
    return *(int*)a - *(int*)b;
}

compare = cmp_int;
int result = compare(&x, &y);
        
// Classic example: qsort (C's generic sort)
int scores[] = {50, 20, 80, 10, 40};

int cmp_desc(const void *a, const void *b) {
    return *(int*)b - *(int*)a;  // descending
}

qsort(scores, 5, sizeof(int), cmp_desc);
// scores is now {80, 50, 40, 20, 10}

// How C fakes vtables (polymorphism via function pointers):
typedef struct {
    void (*draw)(void *self);
    void (*update)(void *self, float dt);
    void (*destroy)(void *self);
} ShapeVTable;

typedef struct {
    ShapeVTable *vtable;  // pointer to function table
    float x, y;
} Shape;

Error Handling

C# — Exceptions

try {
    var data = File.ReadAllText(path);
    Process(data);
}
catch (FileNotFoundException ex) {
    Console.Error.WriteLine(
        $"Not found: {ex.Message}");
}
finally {
    Cleanup();
}

C — Return Codes + errno

#include <errno.h>

FILE *f = fopen(path, "r");
if (f == NULL) {
    // errno is set by fopen
    perror("fopen failed");
    // prints: "fopen failed: No such
    //          file or directory"
    return -1;
}
// process file...
fclose(f);
        
// The "goto cleanup" pattern (widely used in the Linux kernel):
int process_file(const char *path) {
    int result = -1;
    FILE *f = NULL;
    char *buf = NULL;

    f = fopen(path, "r");
    if (!f) goto cleanup;

    buf = malloc(4096);
    if (!buf) goto cleanup;

    if (fread(buf, 1, 4096, f) == 0) goto cleanup;

    // Success path
    result = 0;

cleanup:
    free(buf);        // free(NULL) is safe
    if (f) fclose(f);
    return result;    // 0 = success, -1 = error
}
// This is C's equivalent of try/finally!

File I/O

C# — File Class

// Read all text
string text =
    File.ReadAllText("data.txt");

// Write
File.WriteAllText("out.txt", text);

// Stream reading
using var reader =
    new StreamReader("data.txt");
string? line;
while ((line = reader.ReadLine())
        != null) {
    Console.WriteLine(line);
}

C — FILE* Streams

// Read text file
FILE *f = fopen("data.txt", "r");
if (!f) { perror("fopen"); return; }

char line[256];
while (fgets(line, sizeof(line), f)) {
    printf("%s", line);
}
fclose(f);  // No 'using' -- manual!

// Write text file
FILE *out = fopen("out.txt", "w");
fprintf(out, "Score: %d\n", 42);
fclose(out);

// Binary I/O
FILE *bin = fopen("data.bin", "rb");
int nums[10];
fread(nums, sizeof(int), 10, bin);
fclose(bin);
  • No using statement — you must fclose() every fopen()
  • Mode strings: "r" read, "w" write (truncates), "a" append, "rb"/"wb" binary
  • fprintf/fscanf for formatted I/O, fread/fwrite for binary

Common Patterns

Opaque Types (pimpl)

Hide struct internals from users. Header declares only a pointer.

// db.h -- users see only this:
typedef struct Database Database;
Database *db_open(const char *path);
void db_close(Database *db);

// db.c -- internals hidden here:
struct Database {
    FILE *file;
    int cache_size;
};

Tagged Unions

C's version of discriminated unions / sum types.

typedef enum {
    SHAPE_CIRCLE, SHAPE_RECT
} ShapeType;

typedef struct {
    ShapeType type;
    union {
        struct { float radius; } circle;
        struct { float w, h; } rect;
    };
} Shape;

Arena Allocator

Allocate many objects, free them all at once. Fast and leak-proof.

typedef struct {
    char *buf;
    size_t used, cap;
} Arena;

void *arena_alloc(Arena *a, size_t n) {
    void *p = a->buf + a->used;
    a->used += n;
    return p;
}
void arena_free(Arena *a) {
    a->used = 0; // reset everything!
}

Linked List (by Hand)

No List<T> in C. You build data structures from scratch.

typedef struct Node {
    int value;
    struct Node *next;
} Node;

void push(Node **head, int val) {
    Node *n = malloc(sizeof(Node));
    n->value = val;
    n->next = *head;
    *head = n;
}

Undefined Behavior

"The compiler assumes undefined behavior never happens and optimizes accordingly."

// 1. Signed integer overflow (C# wraps around; C = UB)
int x = INT_MAX;
x + 1;  // UB! Compiler may optimize away checks like "if (x+1 < x)"

// 2. Null pointer dereference
int *p = NULL;
*p = 42;  // In C#: NullReferenceException. In C: UB (anything can happen)

// 3. Buffer overrun
int arr[5];
arr[10] = 1;  // In C#: IndexOutOfRangeException. In C: silent corruption

// 4. Use after free
free(p);
printf("%d", *p);  // May print old value, crash, or launch nethack

// 5. Uninitialized variable
int y;
printf("%d", y);  // Could be 0, could be 42, could be -9999

// 6. Accessing union member that wasn't last written
// 7. Shifting by more than width of type (e.g., 1 << 33 for int)
// 8. Modifying a string literal: char *s = "hello"; s[0] = 'H';

Real-world consequence: The compiler can delete your null check, reorder operations, or remove entire code blocks if it proves that reaching them requires UB. Your "working" debug build may break in release builds with optimizations.

# Debug with gdb
gcc -g -O0 main.c -o app_debug     # compile with debug symbols
gdb ./app_debug                     # start debugger
  (gdb) break main                  # set breakpoint
  (gdb) run                         # run program
  (gdb) print variable              # inspect variables
  (gdb) backtrace                   # show call stack

# Sanitizers (catch bugs at runtime -- like C#'s safety, but opt-in)
gcc -fsanitize=address main.c       # ASan: buffer overflows, use-after-free
gcc -fsanitize=undefined main.c     # UBSan: undefined behavior
gcc -fsanitize=memory main.c        # MSan: uninitialized reads (clang only)

# Valgrind (memory leak detector)
valgrind --leak-check=full ./app

When to Use C

Embedded Systems

Microcontrollers with 2KB RAM. No OS, no runtime. C is often the only option.

OS Kernels & Drivers

Linux, Windows NT kernel, macOS kernel — all written in C. Hardware needs direct memory access.

Game Engines & Graphics

Performance-critical inner loops, GPU interfaces, real-time systems where GC pauses are fatal.

Interpreters & Compilers

CPython, Lua, Ruby — your favorite languages are implemented in C.

Performance Libraries

SQLite, zlib, OpenSSL, curl — used by every language via FFI/bindings.

C# Interop via P/Invoke

Call C libraries from C#: [DllImport("mylib.so")] for native performance where you need it.

When to stay in C#: Web apps, business logic, desktop apps, rapid prototyping — anywhere developer productivity and safety matter more than raw performance.
