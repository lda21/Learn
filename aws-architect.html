AWS Well-Architected Framework

  • Operational Excellence — automate operations, respond to events, learn from failures
  • Security — protect data, systems, and assets with least privilege
  • Reliability — recover from failures, scale to meet demand
  • Performance Efficiency — use resources efficiently as demand changes
  • Cost Optimization — deliver value at the lowest price point
  • Sustainability — minimize environmental impact of workloads

Compute: EC2 & Auto Scaling

  • Instance types: General (t3, m6i), Compute (c6i), Memory (r6i), GPU (p4d)
  • Auto Scaling Groups: scale in/out based on CloudWatch metrics
  • Launch Templates: define AMI, instance type, security groups, user data
  • Spot Instances: up to 90% savings for fault-tolerant workloads
# Auto Scaling Policy
ScalingPolicy:
  PolicyType: TargetTrackingScaling
  TargetTrackingConfiguration:
    PredefinedMetricSpecification:
      PredefinedMetricType: ASGAverageCPUUtilization
    TargetValue: 70.0    # Scale when CPU > 70%

Security: Encryption & Network

  • KMS — managed encryption keys, integrates with all AWS services
  • ACM — free SSL/TLS certificates for ALB, CloudFront
  • Security Groups — stateful firewall at instance level
  • NACLs — stateless firewall at subnet level
  • WAF — web application firewall for ALB, CloudFront, API Gateway
  • GuardDuty — threat detection from VPC flow logs, DNS, CloudTrail

Cost Optimization

1 Use Reserved Instances or Savings Plans for steady workloads (up to 72% off)
2 Use Spot Instances for fault-tolerant workloads (up to 90% off)
3 Right-size instances with AWS Compute Optimizer
4 S3 Lifecycle policies to move data to cheaper tiers
5 Use Cost Explorer and set Budgets with alerts
6 Delete unused EBS volumes, snapshots, and Elastic IPs

Containers: ECS & EKS

ECS (Elastic Container Service)

AWS-native container orchestration. Simpler than Kubernetes. Best for AWS-only shops.

Fargate = serverless containers (no EC2)

EKS (Elastic Kubernetes Service)

Managed Kubernetes. Use if you need K8s portability or ecosystem tools (Helm, Istio).

EKS Fargate = serverless K8s pods

Architecture Best Practices

1 Design for failure — everything fails, plan for it
2 Decouple components with SQS, SNS, EventBridge
3 Use managed services over self-managed (RDS over self-hosted DB)
4 Implement least privilege IAM everywhere
5 Encrypt everything — at rest and in transit
6 Use Infrastructure as Code (Terraform / CloudFormation)
7 Deploy across multiple AZs minimum
8 Automate everything — scaling, deployments, recovery
